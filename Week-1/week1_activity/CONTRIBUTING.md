# Contributing Guide

Thank you for contributing to this project!

This document outlines our branch, pull request, and review conventions.

## Branching Strategy
- main -> Production ready
- dev -> Intermidiate step between Feature* and main
- Feature_Stefano -> Changes made by Stefano
- Feature_Paola -> Changes made by Paola
- Feature_Axel -> Changes made by Axel

## PR Rules
- Use pull requests only
- At least 1 reviewer required

## Commit Style
- Feat: add new feature
- Modify: modify what documents
- fix: bug fix
